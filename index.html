JavaScript、CSS、SVG 做網頁動畫時，各自在控制什麼？

CSS 擅長描述「樣式如何隨時間變化」，SVG 擅長描述「向量圖形如何變形或沿路徑運動」， JavaScript 擅長描述「動畫如何回應資料、事件、物理與使用者狀態」。

一句話區分

CSS 是樣式時間軸，SVG 是向量圖形語言，JavaScript 是程式控制層。實務上它們常被混用： 用 SVG 畫精準圖形，用 CSS 做轉場，用 JavaScript 控制狀態與互動。

選型優先順序

先問動畫是否只改外觀。若是，優先 CSS。若圖形本身很重要，優先 SVG。 若需要讀取資料、即時互動、遊戲或物理計算，再用 JavaScript。

01 · Rendering Model

動畫原理

CSS

CSS：宣告式關鍵影格

透過 `@keyframes`、`animation`、`transition` 宣告元素屬性在時間軸上的變化。 瀏覽器可自行排程，常見的 `transform` 與 `opacity` 動畫能避開昂貴的版面重排。

  • 適合 hover、開合、載入狀態、頁面微互動。
  • 程式邏輯少，維護成本低。
  • 不擅長複雜條件、碰撞偵測、資料驅動流程。
SVG

SVG：向量圖形與圖形 DOM

SVG 用 XML 元素描述線、圓、路徑、遮罩、漸層等向量圖形。動畫可用 CSS、SMIL 動畫元素，或 JavaScript 操作 SVG DOM。

  • 縮放不失真，適合圖示、圖解、資料視覺化。
  • 可針對路徑、節點、描邊、遮罩做細緻動畫。
  • 大量複雜節點可能造成 DOM 成本。
JS

JavaScript：逐幀計算與 API 控制

JavaScript 可用 `requestAnimationFrame()` 在瀏覽器重繪前更新狀態，也可用 Web Animations API 取得與控制動畫物件，例如暫停、調速、同步多個動畫。

  • 適合互動模擬、遊戲、圖表、資料驅動動畫。
  • 可整合事件、時間、速度、物理與遠端資料。
  • 若每幀做太多 DOM 或 layout 操作，效能容易下降。

02 · Comparison

差異比較

面向 CSS 動畫 SVG 動畫 JavaScript 動畫
核心語意 樣式屬性隨時間改變 向量圖形、路徑、描邊與形狀變化 程式根據狀態逐幀更新
控制方式 宣告式，瀏覽器排程 宣告式或 DOM 操作 命令式或透過 Web Animations API
互動能力 適合簡單狀態與媒體查詢 可對圖形元素加事件與樣式 最強，可讀資料、事件、感測與時間
效能重點 優先 animating `transform`、`opacity` 控制節點數，避免過度複雜濾鏡 減少每幀 DOM 讀寫，批次更新
典型用途 按鈕、卡片、選單、loading、頁面轉場 logo、icon、流程圖、科學圖解、資料視覺化 遊戲、模擬器、互動圖表、滾動敘事

03 · Design Roles

在網頁設計中的強項

介面層：CSS 最強

元件 hover、focus、active、enter/exit transition、RWD 狀態切換都適合 CSS。 它讓互動回饋靠近樣式系統，設計一致性最好維護。

圖形層：SVG 最強

需要清楚呈現圖形結構時，SVG 比 bitmap 更適合。像流程箭頭、結構示意、 曲線、節點、遮罩與描邊動畫，都能保留語意與可縮放性。

行為層：JavaScript 最強

當動畫必須跟使用者輸入、資料、時間差、物理量或多個元件同步時， JavaScript 是控制中心。CSS 與 SVG 可負責呈現，JS 負責決策。

混合層：實務最常見

例如互動式教材可用 SVG 畫科學圖形，用 CSS 做狀態轉場，用 JavaScript 根據滑桿、測驗或模擬數值更新畫面。

05 · Decision Guide

實務選擇建議

  1. 只改視覺狀態：用 CSS transition 或 keyframes。
  2. 需要清楚、可縮放的圖形：用 SVG 畫，再用 CSS 或 SVG 動畫。
  3. 需要根據資料或輸入即時變化：用 JavaScript 控制，必要時搭配 Canvas、SVG 或 Web Animations API。
  4. 需要可維護：把「畫什麼」與「何時變化」分開。SVG/CSS 負責呈現，JavaScript 負責狀態。
